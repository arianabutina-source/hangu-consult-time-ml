# Frontend — React + Vite

The dashboard and prediction UI for the Optimised Scheduling Tool. Talks to the FastAPI
backend via `fetch()`; the dashboard's charts read static JSON exported by
`scripts/export_dashboard_data.py` (no live model calls needed to browse results).

## Routes

| Path | Page |
|---|---|
| `/` | Home — motivation, hero, benefits |
| `/dashboard` | "Appointment Length Calculator" — model performance, results, feature importance |
| `/predict/classification` | Live prediction form (long vs. short consultation) — shows every model's prediction |
| `/predict/regression` | Live duration-estimate form (minutes) — shows every model's prediction |
| `/scheduling-script` | Patient communication script |
| `/pricelist` | Service pricing tables |
| `/team` | Clinic team |

## Setup

```bash
cd frontend
npm install
npm run dev
```

Visit `http://localhost:5173`. Requires the backend running locally at
`http://localhost:8000` (see [`../backend/README.md`](../backend/README.md)) — or point
`VITE_API_BASE_URL` at any reachable backend, including the deployed one.

## Environment Variables

| Variable | File | Purpose |
|---|---|---|
| `VITE_API_BASE_URL` | `.env.development` (local), Vercel dashboard (production) | Base URL of the FastAPI backend |

`.env.example` documents the variable; `.env.development` is committed with the local
default (`http://localhost:8000`).

## Dashboard Data

The dashboard's charts (confusion matrix, ROC curve, feature importance, residuals) read
static JSON from `public/data/`, generated by `scripts/export_dashboard_data.py` at the
repo root. After retraining the models, refresh this data with:

```bash
python -m scripts.export_dashboard_data          # from the repo root
./frontend/scripts/sync-dashboard-data.sh        # copies ml/artifacts/dashboard/* here
```

## Scripts

```bash
npm run dev      # start the dev server (HMR)

npm run lint      # oxlint
```

## Deployment

Deployed on Vercel. To deploy manually:

```bash
npx vercel --prod
```

Requires `VITE_API_BASE_URL` set in the Vercel project's environment variables (both
Production and Preview), pointing at the live backend. The backend's `ALLOWED_ORIGINS`
must include this project's Vercel domain for CORS to succeed — see the root
[README](../README.md#deployment).
